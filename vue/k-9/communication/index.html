组件通信：父子通信、provide/inject

用课程主题同步展示跨层级依赖注入的基本方式。

案例演示

案例交互加载中...

关键代码

<script setup lang="ts">
import { computed, defineComponent, h, inject, provide, ref, type Ref } from 'vue'

const theme = ref('青色')
provide('courseTheme', theme)

const ThemePreview = defineComponent({
  setup() {
    const injectedTheme = inject<Ref<string>>('courseTheme', ref('默认主题'))
    const message = computed(() => `子组件读取到主题：${injectedTheme.value}`)

    return () => h('p', message.value)
  },
})
</script>

<template>
  <div class="demo-card">
    <h3>课程主题同步</h3>
    <select v-model="theme">
      <option>青色</option>
      <option>绿色</option>
      <option>红色</option>
    </select>
    <ThemePreview />
  </div>
</template>

原理

父子之间优先使用 props 和 emits；当数据需要跨过多层组件传递，并且它更像上下文能力时，可以由上层 provide，再由后代 inject 读取。

处理流程

  1. 上层组件维护 theme，并通过 provide 暴露给后代。
  2. 后代组件通过 inject 获取同一个响应式主题。
  3. 用户切换主题时，依赖该主题的后代视图自动更新。

注意事项

  • provide/inject 适合主题、表单上下文、组件库配置等稳定上下文。
  • 频繁变化且跨页面共享的业务状态更适合 Pinia。
  • 大型项目建议使用 Symbol 作为注入 key，避免字符串冲突。
