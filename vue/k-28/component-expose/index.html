组件公开接口：defineExpose、useTemplateRef

用课程搜索面板演示父组件通过模板引用调用子组件明确公开的聚焦与清空能力。

关键代码

<script setup lang="ts">
import { useTemplateRef } from 'vue'
import K28SearchPanel from './K28SearchPanel.vue'

const searchPanel = useTemplateRef<InstanceType<typeof K28SearchPanel>>('searchPanel')

function focusChildSearch() {
  searchPanel.value?.focusSearch()
}

function clearChildSearch() {
  searchPanel.value?.clearSearch()
}
</script>

<template>
  <div class="demo-card expose-demo">
    <div>
      <p class="demo-kicker">受控的组件实例能力</p>
      <h3>课程检索控制台</h3>
    </div>

    <K28SearchPanel ref="searchPanel" />

    <div class="button-row">
      <button @click="focusChildSearch">聚焦子组件输入框</button>
      <button class="secondary" @click="clearChildSearch">清空并重新聚焦</button>
    </div>

    <p class="demo-note">
      父组件只能访问 defineExpose 明确公开的方法，子组件内部状态仍保持封装。
    </p>
  </div>
</template>

原理

script setup 组件默认是封闭的，父组件不能任意访问其内部变量。子组件通过 defineExpose 明确公开少量命令式能力，父组件再用 useTemplateRef 获取类型安全的组件引用。

处理流程

  1. 子组件维护输入值和内部 DOM 引用。
  2. 通过 defineExpose 只公开 focusSearch 与 clearSearch 方法。
  3. 父组件使用 useTemplateRef 获取模板中的子组件实例。
  4. 用户点击外部工具按钮时，父组件调用公开方法完成聚焦或清空。

注意事项

  • 优先使用 props 和 emits 进行声明式通信，只在聚焦、滚动、播放等命令式场景使用组件引用。
  • defineExpose 应保持接口最小，不要把全部内部状态暴露给父组件。
  • useTemplateRef 是 Vue 3.5 提供的模板引用 API，引用值在组件挂载前为 null。
