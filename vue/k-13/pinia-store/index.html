🌰13 · 状态管理

Pinia 状态管理：store、getter、action

用课程购物车展示全局 store 的状态、派生值和业务动作。

案例演示

案例交互加载中...

关键代码

<script setup lang="ts">
import { useCartStore } from '../stores/cart'

const cart = useCartStore()
</script>

<template>
  <div class="demo-card">
    <h3>课程购物车</h3>
    <ul>
      <li v-for="item in cart.items" :key="item.id">
        {{ item.name }} x {{ item.count }}
      </li>
    </ul>
    <strong>总价：{{ cart.total }} 元</strong>
    <button @click="cart.addCourse">再买一份课程</button>
  </div>
</template>

原理

Pinia 把跨组件共享的业务状态集中到 store。state 保存数据，getter 表达派生结果，action 封装修改流程，让组件不用知道状态修改细节。

处理流程

  1. 组件通过 useCartStore 获取购物车 store。
  2. 列表读取 cart.items，总价读取 cart.total getter。
  3. 点击按钮调用 cart.addCourse，由 action 修改课程数量。

注意事项

  • 只有跨组件或跨页面共享的状态才值得放入 Pinia。
  • 复杂修改流程放进 action，组件只表达用户意图。
  • store 的 getter 和 action 很适合写单元测试。
