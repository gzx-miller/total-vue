🌰6 · 用户输入

表单处理：v-model、修饰符、基础校验

用活动报名表展示双向绑定、输入清洗和提交条件控制。

案例演示

案例交互加载中...

关键代码

<script setup lang="ts">
import { computed, reactive } from 'vue'

const form = reactive({
  name: '',
  email: '',
  agree: false,
})

const canSubmit = computed(() => form.name.trim().length >= 2 && form.email.includes('@') && form.agree)
</script>

<template>
  <form class="demo-card" @submit.prevent>
    <h3>活动报名表</h3>
    <input v-model.trim="form.name" placeholder="姓名，至少 2 个字符" />
    <input v-model.trim="form.email" placeholder="邮箱" />
    <label><input v-model="form.agree" type="checkbox" /> 同意接收课程通知</label>
    <button :disabled="!canSubmit">提交报名</button>
    <p>{{ canSubmit ? '表单可以提交' : '请补全信息' }}</p>
  </form>
</template>

原理

v-model 把表单控件和响应式状态连接成双向关系；修饰符在同步前处理常见输入细节；校验逻辑通常用 computed 表达，让按钮状态和提示文案自动更新。

处理流程

  1. 用户填写姓名、邮箱并勾选同意项。
  2. v-model.trim 把输入同步到 form，同时去掉首尾空格。
  3. canSubmit 根据表单状态计算是否允许提交。

注意事项

  • 前端校验用于即时反馈，真实提交仍必须依赖服务端校验。
  • 表单字段多且属于同一业务对象时，reactive 对象更便于组织。
  • 不要只通过禁用按钮表达错误，必要时应给出明确提示。
