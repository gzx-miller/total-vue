原理

某些内容只能在浏览器渲染：图表库直接操作 DOM、浏览器 API（window/navigator）、动态内容（时间/随机数）。Nuxt 提供多种方式处理：ClientOnly 组件包裹仅客户端的内容、false 条件判断、.client.ts 文件名后缀让插件只在客户端加载、onMounted 中赋值浏览器特定数据。

处理流程

  1. 遇到不兼容 SSR 的库，用 ClientOnly 包裹并提供 fallback。
  2. 需要浏览器 API 时，用 false 判断或 onMounted 延迟赋值。
  3. 第三方库 SSR 报错时，在 plugins/ 中创建 .client.ts 文件仅客户端注册。

注意事项

  • ClientOnly 的 fallback slot 在 SSR 时显示，客户端加载完成后替换。
  • false 是编译时常量，不会增加运行时开销。
  • 大量使用 ClientOnly 会削弱 SSR 的 SEO 优势，应控制使用范围。
